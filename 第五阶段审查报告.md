## 已修复问题

### 1. 桌面端生产态未串起本地 API 的问题修复

原问题：

- Electron 生产态直接加载本地构建文件
- 前端 API Client 依赖相对路径，file:// 场景下无法稳定命中本地 API
- 生产态没有本地 API 启动逻辑，也没有可用的桌面启动脚本

修复结果：

- 桌面主进程在非开发态会自动拉起本地 API 进程
- 增加健康检查等待逻辑，API 就绪后再加载界面
- API Client 增加 `file://` 场景下的本地基址回退
- 新增 `desktop:start` 启动脚本，形成 `ui build -> electron -> 本地 API` 的启动闭环

影响：

- 桌面端生产态不再只是开发态模拟
- “前端构建产物 + 本地 API + 启动脚本”的最小闭环已成立

### 2. 搜索页状态与 URL 不一致的问题修复

原问题：

- 页面初始化时会从 URL 读取筛选条件
- 但 URL 变化后只有 query 和 mode 会同步，sources 和 tags 会残留在组件 state 中
- 会导致显示的地址栏与真实搜索条件不一致

修复结果：

- 搜索页现在以 URL 参数为提交后的唯一状态源
- sources、tags、mode、timeRange、page 都会从 URL 统一解析
- 全局搜索跳转、浏览器导航和搜索表单提交后，筛选条件会保持一致

影响：

- 搜索结果不会再被隐藏的旧筛选条件悄悄污染
- 页面状态与 URL 状态重新对齐

### 3. 搜索页与标签页功能完整性问题修复

原问题：

- 搜索页缺少时间筛选与分页
- 标签详情页缺少排序与分页
- 标签详情接口虽然存在分页能力，但前端没有消费

修复结果：

- 搜索页增加时间范围筛选
- 搜索页结果支持分页
- 标签页增加分类切换、排序和分页
- API Client 补齐搜索和标签接口的分页参数封装

影响：

- 搜索和标签主链路更接近第五阶段计划定义
- 数据量增长后仍能维持可用体验

### 4. 详情页未提供显式“标记已读按钮”的问题修复

原问题：

- 详情页在挂载时自动记录历史
- 没有显式的“标记已读”按钮

修复结果：

- 详情页改为提供显式“Mark as read”按钮
- 点击后才写入浏览历史
- 详情页同时根据历史状态更新按钮文案

影响：

- 已读动作从隐式变为显式
- 与阶段计划中的交互要求一致

### 5. 第五阶段测试覆盖不足的问题修复

原问题：

- 仅有首页 smoke test
- 缺少搜索页、详情页、用户中心和 API Client 的专项测试

修复结果：

- 新增搜索页测试
- 新增详情页测试
- 新增用户中心测试
- 新增 API Client 测试
- 旧有首页、API、搜索测试仍保持通过

影响：
