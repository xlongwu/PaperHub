# 第六阶段 Windows 打包与发布任务清单

配套执行文档：

- `第六阶段Windows实机执行清单.md`

## 目标

本清单面向第六阶段执行，目标不是继续扩功能，而是把 PaperHub 收敛成一个可以在 **Windows** 上安装、启动、完成主链路验证并具备首版发布条件的桌面应用。

本清单完成后，团队应能做到：

- 在 Windows 环境生成可分发安装包
- 在干净机器上完成安装与启动
- 验证本地 API、首页、搜索、详情、标签、用户中心的主链路
- 输出首版发布候选与验收记录

---

## 范围与边界

### 本清单覆盖

- Windows 安装包构建
- Windows 运行环境适配
- Windows 主链路验收
- 发布说明与回滚说明
- 发布前自动化与手工门禁

### 本清单不覆盖

- macOS 安装包
- 自动更新正式接入
- 应用商店分发
- 云同步、多用户、额外功能开发

---

## 执行原则

1. 先保证 Windows 可安装可运行，再谈多平台。
2. 先保证无 blocker 的首版交付，再做体验 polish。
3. 先补发布闭环，再做非阻断优化。
4. 所有任务都必须带验收结果，不接受“理论可行但未验证”。

---

## 任务拆解

## 1. 打包路线定板

### 1.1 统一打包方案

- [ ] 明确继续使用 Electron 路线，不在第六阶段切换 Tauri
- [ ] 选定 Windows 打包工具，优先在以下二选一中定板：
  - `electron-builder`
  - `electron-forge`
- [ ] 确定首版安装包格式，优先以下二选一：
  - `NSIS .exe`
  - 便携版 `portable .exe`

验收标准：

- 输出一条明确结论：使用什么打包工具、产出什么格式、为什么

阻断条件：

- 若打包工具迟迟未定，第六阶段后续任务全部会被阻塞

---

## 2. Windows 运行前提补齐

### 2.1 应用基础元数据

- [ ] 设置应用名、版本号、产品名
- [ ] 设置 Windows `appId`
- [ ] 准备应用图标（`.ico`）
- [ ] 确认安装目录、用户数据目录、日志目录在 Windows 下可写

验收标准：

- 本地运行时窗口标题、应用名、目录行为符合预期
- 应用首次启动不会因路径问题失败

### 2.2 本地 API 进程启动验证

- [ ] 验证 Electron 主进程在 Windows 下可以拉起 Node 本地 API
- [ ] 验证健康检查等待逻辑在 Windows 下有效
- [ ] 验证 API 退出时桌面端的错误提示和退出逻辑
- [ ] 验证重复打开应用不会造成端口冲突

验收标准：

- 在 Windows 上从桌面入口启动，能稳定进入首页
- 启动失败时有可理解的报错，不是静默白屏

阻断条件：

- API 无法随桌面端稳定拉起
- 启动阶段频繁出现端口冲突或无响应

---

## 3. Windows 打包配置落地

### 3.1 安装包构建配置

- [ ] 增加 Windows 打包配置文件或 `package.json` 构建段
- [ ] 配置入口文件、构建产物目录、静态资源打包策略
- [ ] 确认 `ui build` 与 Electron 主进程产物能被打包工具正确收集
- [ ] 配置安装包输出目录与命名规范

建议产物命名：

- `PaperHub-0.1.0-win-x64-setup.exe`
- `PaperHub-0.1.0-win-x64-portable.exe`

验收标准：

- 一条命令可以完成 Windows 安装包构建
- 构建完成后产物位置和命名可预测

### 3.2 资源与依赖打包校验

- [ ] 验证 `dist-ui` 被正确包含
- [ ] 验证 Electron 主进程脚本被正确包含
- [ ] 验证生产态启动时需要的 Node 依赖被正确包含
- [ ] 验证 SQLite、`better-sqlite3`、`sqlite-vec` 在 Windows 构建环境可用

验收标准：

- 安装包解出或安装后，启动不出现“找不到文件”或“原生模块不可用”

阻断条件：

- 原生依赖在 Windows 下无法打包或运行
- 安装包内缺少 UI、API 或关键运行资源

---

## 4. Windows 安装与启动验收

### 4.1 干净环境安装验证

- [ ] 在 Windows 干净环境执行安装
- [ ] 验证安装流程能完成
- [ ] 验证首次启动能进入应用
- [ ] 验证卸载流程无明显异常

验收标准：

- 用户不需要手工执行额外命令即可安装并启动

### 4.2 首次启动验收

- [ ] 启动应用
- [ ] 验证首页加载
- [ ] 验证本地数据库初始化
- [ ] 验证日志目录生成
- [ ] 验证用户数据目录生成

验收标准：

- 首次启动无白屏、无卡死、无明显报错弹窗

---

## 5. Windows 主链路手工测试

### 5.1 首页

- [ ] 热点推荐可见
- [ ] 个性化推荐可见
- [ ] 最近更新列表可见
- [ ] 导航跳转正常

### 5.2 搜索页

- [ ] 可输入关键词并提交
- [ ] 来源筛选可生效
- [ ] 标签筛选可生效
- [ ] 时间范围筛选可生效
- [ ] 分页可切换
- [ ] 调研报告面板可正常显示或降级为空态

### 5.3 详情页

- [ ] 详情页可打开
- [ ] 标题、作者、摘要、标签、来源信息可见
- [ ] “Mark as read” 可写入历史
- [ ] 收藏动作可写入收藏夹
- [ ] 原文链接可正常打开浏览器

### 5.4 标签页

- [ ] 标签总览可见
- [ ] 分类切换可用
- [ ] 标签详情可打开
- [ ] 排序可切换
- [ ] 分页可切换

### 5.5 用户中心

- [ ] 兴趣标签可编辑并保存
- [ ] 用户记忆可查看并手动刷新
- [ ] 浏览历史可看到已读条目
- [ ] 收藏夹可看到收藏条目并可移除
- [ ] 偏好设置重启后仍保留

验收标准：

- 五个页面主链路全部通过
- 不存在阻断使用的 P0 缺陷

### 7.1 必备文档

- [ ] Windows 打包说明
- [ ] 首版发布说明
- [ ] 已知问题清单

---

## 8. 发布阻断项清单

以下问题任意一项存在，都不能发布 Windows 首版：

- [ ] 安装包无法生成
- [ ] 安装后无法启动
- [ ] 启动后本地 API 无法连通
- [ ] 首页、搜索、详情、标签、用户中心任一主链路不可用
- [ ] 收藏、已读、偏好保存存在数据丢失
- [ ] 存在明显白屏、闪退或高频卡死
- [ ] 发布文档缺失

---

## 9. 发布通过标准

满足以下条件即可视为 Windows 首版可发布：

- [ ] Windows 安装包已生成
- [ ] Windows 干净环境安装通过
- [ ] 五个核心页面主链路手工验收通过
- [ ] 自动化门禁全部通过
- [ ] 已知问题中没有 P0 阻断项
- [ ] 发布说明、回滚说明、验收记录齐备

建议发布结论格式：

- `可发布`
- `有条件可发布`
- `不可发布，需回修`

---

## 10. 建议执行顺序

按以下顺序推进，避免互相阻塞：

1. 定板 Windows 打包工具与安装包格式
2. 补齐 Windows 打包配置
3. 验证 Windows 下本地 API 启动与依赖可用
4. 生成首个安装包
5. 执行干净环境安装与启动验收
6. 执行五页面主链路手工测试
7. 跑自动化门禁
8. 整理发布说明、已知问题与验收记录
9. 给出首版发布结论

---

## 11. 下一阶段预留项

Windows 首版完成后，下一步建议顺序：

1. macOS 打包与兼容性验证
2. Windows 安装包签名
3. 自动更新机制
4. 更完整的桌面端 E2E
5. 多平台发布流水线
