## 已修复问题

### 1. 第六阶段 E2E 验证链路在当前环境不可落地的问题修复

原问题：

- 原第六阶段验证方案依赖浏览器级 Playwright 链路
- 当前执行环境对 Node 侧 localhost 与浏览器自动化存在限制
- 导致 `pnpm test:e2e` 无法在本环境形成稳定、可复跑的发布前门禁

修复结果：

- 将第六阶段 `test:e2e` 收敛为可在当前环境执行的 release smoke
- 由 `scripts/run-phase6-e2e.mjs` 串联 `ui:build -> 固定夹具 seed -> vitest release smoke`
- 新增并稳定化 `src/__tests__/phase6-release.test.tsx`，直接覆盖真实 API 逻辑与真实本地数据库状态

影响：

- 第六阶段自动化门禁不再依赖当前环境下不可稳定执行的浏览器自动化
- 发布前验证重新具备可重复、可追踪、可集成的执行路径

### 2. 第六阶段 release smoke 断言与运行环境不一致的问题修复

原问题：

- 测试数据目录默认会落到 `~/PaperHub`，在当前沙箱下触发只读路径错误
- 首页存在重复文案命中，测试选择器不稳定
- 搜索/详情/个人中心链路断言与预置历史、收藏状态冲突
- 标签页分页断言与真实排序、加载时序不一致

修复结果：

- 测试显式固定 `PAPERHUB_DATA_DIR` 到工作区 `.tmp`
- 首页改为使用稳定的多命中断言
- 搜索链路改为验证未预置已读/收藏的真实文档
- 个人偏好保存断言补齐异步回填与持久化校验
- 标签页补齐分页等待，并按真实第 2 页结果修正断言

影响：

- 第六阶段 release smoke 现在可稳定通过
- 首页、搜索、详情、标签、用户中心的关键回归链路重新获得保护

### 3. 第六阶段运行文档与实际验证链路不一致的问题修复

原问题：

- `docs/phase6-runbook.md` 仍按旧的 Playwright 浏览器链路描述 `pnpm test:e2e`
- 文档仍将 `test:e2e` 标记为当前环境阻塞项
- 与现已落地的可执行 release smoke 事实不一致

修复结果：

- 更新第六阶段 runbook
- 将 `test:e2e` 描述同步为当前真实执行方式
- 将验证状态同步为已通过
- 保留真正未完成的 Windows 打包主机依赖作为剩余阻塞项

影响：

- 第六阶段文档、脚本和验证事实重新对齐
- 后续执行者不会再被过时说明误导
